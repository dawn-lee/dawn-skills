# Prompt Guide - 文生视频/图生视频提示词指南

在使用万相进行AI视频内容创作时，可运用**结构化提示词公式**，提升生成视频的质量与艺术表现力。

---

## 提示词公式

提示词（prompt）是指**正向提示词**，用来描述视频中所包含的画面内容和运动过程，描述越准确、越丰富，生成视频的质量越高。

---

## 基础公式

适用于初次尝试AI视频的新用户，简单自由的提示词可生成更具有想象力的视频。

**提示词 = 主体 + 场景 + 运动**

- **主体:** 主体是视频内容的主要表现对象，可以是人、动物、植物、物品或非物理真实存在的想象物体。
- **场景:** 场景是主体所处的环境，包含背景、前景，可以是物理存在的真实空间或想象出来的虚构场景。
- **运动:** 运动包含主体的具体运动和非主体的运动状态，可以是静止、小幅度运动、大幅度运动、局部运动或整体动势。

---

## 进阶公式

适用于有一定AI视频使用经验的用户，添加更丰富细致的描述可有效提升视频质感、生动性与故事性。

**提示词 = 主体（主体描述）+ 场景（场景描述）+ 运动（运动描述）+ 美学控制 + 风格化**

- **主体描述:** 对主体外观特征细节的描述，如"一位身着少数民族服饰的黑发苗族少女"、"一位来自异世界的飞天仙子，身着破旧却华丽的服饰，背后展开一对由废墟碎片构成的奇异翅膀"。
- **场景描述:** 对主体所处环境特征细节的描述。
- **运动描述:** 对运动特征细节的描述，包含运动的幅度、速率和运动作用的效果，如"猛烈地摇摆"、"缓慢地移动"、"打碎了玻璃"。
- **美学控制:** 包含光源、光线环境、景别、视角、镜头、运镜等，详见下方提示词词典。
- **风格化:** 对画面风格语言的描述，如"赛博朋克"、"勾线插画"、"废土风格"，详见下方提示词词典。

---

## 图生视频公式

图像已经确定了主体、场景与风格，因此提示词主要描述动态过程及运镜需求。

**提示词 = 运动 + 运镜**

- **运动描述:** 结合图像中的元素描述其动态过程，如奔跑、打招呼，可通过形容词控制动态程度与速度，如"快速地"、"缓慢地"。
- **运镜:** 通过提示词如"镜头推进"、"镜头左移"控制，若希望镜头不要发生变化，可以通过"固定镜头"来强调。

---

## 声音公式（wan2.7/wan2.6/wan2.5）

基于原生音频能力，增加对人声、音效与背景音乐的描述。

**提示词 = 主体 + 场景 + 运动 + 声音描述（人声/音效/背景音乐）**

声音描述是视频中声音要素及其随时间变化的描述，主要包括人声、音效、BGM三类。

### 人声

**人声 = 角色说话的内容 + 情绪 + 语调 + 语速 + 音色 + 口音**

> 示例：一个男人在讲脱口秀，他说道："好好学习，天天向上"，语气轻松，语速适中，声音清亮，美式英文。

### 音效

**音效 = 音源材质 + 行为 + 环境音**

> 示例：一个玻璃小球从桌面掉在木质地面上，发出"砰"的声音，室内安静环境。

### 背景音乐

**背景音乐 = 背景音乐/配乐 + 风格**

> 示例：雨夜，阴森窄小的走廊，尽头有一扇窗户，配有悬疑风格背景音乐。

## 多镜头公式（wan2.7/wan2.6）

万相2.7和万相2.6模型支持生成多镜头连贯叙事视频。

**提示词 = 总体描述 + 镜头序号 + 时间戳 + 分镜内容**

- **总体描述:** 简要概述整个视频内容，说明故事主题、叙述风格、主要情感或核心事件。
- **镜头序号:** 为每个镜头分配编号，如"镜头1 xxxx，镜头2 xxxx"。
- **时间戳:** 标明每个镜头在视频中的具体时间范围，如"镜头1[0-3秒]"。
- **分镜内容:** 详细描述每个分镜头中主要角色或物体的具体行为。

**示例:**

```
这个故事以第三人称视角，讲述了一个关于放弃与重拾希望的短剧。
第1个镜头[0-3秒]一个男孩在操场的角落独自坐着，低头望着手中的信纸，随后轻轻叹气，眼神中透露出迷茫。
第2个镜头[4-6秒]硬切转场，固定机位，聚焦于男孩的眼睛，泪光闪烁，带着失落和无助。
第3个镜头[7-10秒]硬切转场，场景转至一间简朴的教室。一个女孩眼神温和而坚定，穿着朴素的衣着，面带温和而坚定的笑容，走到男孩的身边安慰他。
```

**单镜头说明:** wan2.7模型不再支持 `shot_type` 指定单镜头/多镜头，而是由模型结合提示词进行发挥。如果想控制多镜头，参考上述提示词写法；如果想控制一镜到底，则中文写为"生成单镜头"，英文写为"Generate single shot."

## 参考生视频公式（wan2.7）

万相2.7模型支持参考图像或视频生成视频。

**提示词 = 参考指代 + 动作 + 场景 + 台词（可选）+ 背景音乐（可选）**

- **参考指代:** 使用"图n"或"视频n"在提示词中指代参考文件，n为序号。图和视频分别计数。
- **动作:** 描述主体或其他元素的运动状态。
- **场景:** 主体所在的环境。
- **台词:** 主体的说话内容。若未描述台词，模型会自由发挥。若想控制不要有台词，中文写为"无台词"。
- **背景音乐:** 不描述时模型自行发挥。不想要背景音乐，中文写为"无背景音乐"。

**示例:**

```
视频2抱着图3在咖啡厅里弹奏一支舒缓的美式乡村民谣，视频1笑着看着视频2。
```

其中，视频1指代的参考视频是一个女性，视频2指代的参考视频是一个男性，图3指代的参考图像是一把吉他。

## 提示词优化技巧

### 1. 自动优化: 启用 prompt_extend 参数

在调用 API 时可启用 `prompt_extend` 参数，系统将自动对输入的正向提示词进行智能优化与改写。该功能对简短、宽泛的提示词（如"小猫在雪地里玩雪球"），提升效果较好。

### 2. 通过提示词公式生成提示词

当需要精准控制画面细节、风格或复杂动态时，推荐采用结构化提示词公式进行编写。可以将上述公式作为系统消息（system message）提供给大语言模型（如 qwen3.6-plus），生成结构清晰的提示词。

| 类型 | 关键词 | 示例 |
|------|--------|------|
| 单人说话 | 说话内容 + 情绪 + 语调 | "她用急促、恐慌的普通话说道：'我现在只能跟随溃败的魏军一起逃跑。'" |
| 多人对话 | 对话内容 + 角色描述 | "她轻启唇瓣，低声呢喃：'这一刻，像梦一样。' 他凝视着她，回应道：'但它是真的。'" |
| 音色 | 音色描述 | "用童音低声说道：'别怕，我会保护你的。'" |
| 方言 | 方言类型 | "用地道的京腔说道：'哟，吃了么您呐？'" |
| 唱歌 | 演唱描述 | "深情地对着麦克风演唱着中文歌曲" |
| 脚步声 | 材质 + 行为 | "金属足部敲击地面发出'咔嗒'节奏声" |

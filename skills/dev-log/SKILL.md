# Dev Log Skill

将编码会话的改动记录到项目的 `DEVELOPMENT_LOG.md` 文件中。
每次会话生成一条顺序编号的条目，包含改动文件、修改内容、遇到的问题，最新条目置于文件顶部。

## 记录时机

**手动触发**：用户明确要求记录时立即执行（触发词见技能描述）。

**主动询问**：当本会话已产生文件变更，且任务到达自然结束点（用户表示「做完了」「就这样」等，或会话即将结束）时，主动询问一次：
> 「改了挺多了，要不要记录一下这次的改动？」

用户确认（「好」「记」「yes」「record」等）后执行；用户拒绝则跳过，同一次会话不再重复询问。若用户已手动触发过记录，不再主动询问。

## 记录格式

每条记录使用以下结构（最新在前）。新建文件时，文件开头为标题 `# Development Log`、说明行 `AI-assisted development change history.`，再接 `---` 分隔线，然后是首个 Session 条目。

```markdown
## Session #N - YYYY-MM-DD HH:MM

**需求**：
{本次会话的目的，一两句话}

**改动文件**：
- `path/to/file` - {新增|修改|删除}, {改动简述}

**变更摘要**：
{本次改动的整体说明}

**遇到的问题**：
- {问题及解决方式}
```

示例（已填好）：

```markdown
## Session #3 - 2026-07-13 14:20

**需求**：
修复 dev-log skill 的若干问题并整理记录格式。

**改动文件**：
- `skills/dev-log/SKILL.md` - 修改, 收窄自动询问范围、修复占位符、合并模板

**变更摘要**：
将自动询问从「任何文件变更」收窄到「任务自然结束点」；删除 body 中与 description 重复的触发词；合并记录格式与新建文件模板并补示例。

**遇到的问题**：
- `{baseDir}` 为未替换占位符，改为 cwd
```

## 记录流程

1. 扫描本次会话中的所有文件变更（新增/修改/删除）及关键命令执行，整理改动清单。

2. 确认需求描述。上下文已明确则直接使用，否则向用户确认。

3. 按下文「文件位置」找到 `DEVELOPMENT_LOG.md`。文件不存在则按「记录格式」模板新建（首行加标题与 `---`）。

4. **去重检查**：记录前先在文件中检索本次会话是否已写入过条目（对照对话历史中已写入的 Session 编号）。若已记录，则在该条目下追加段落并标注 `（续）`，不新建编号；否则取上一个编号 +1 新建条目。

5. 在文件顶部 `---` 分隔线之后插入新条目（最新在前），保留原有内容不变。

## 文件位置

按以下顺序查找 `DEVELOPMENT_LOG.md`：
1. 当前项目根目录
2. 当前工作目录（`cwd`）
3. 无法确定时询问用户
